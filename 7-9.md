- **观测压缩**：环境返回的观测太长时，也压缩一下，替换掉原始观测。

但问题在于，压缩器靠自己的先验知识决定保留什么，没法保证关键信息（比如因果关系、状态变化、前置条件、决策线索）被保留下来。agent 的上下文本质上是个世界模型，信息种类杂、动态性强，压缩必须考虑到这一点。

**优化目标。** 我们要优化压缩器的参数（模型权重 + 压缩准则），目标是最大化任务奖励 + 最小化上下文成本（用 λ 调节两者的权重）。

**3.3 优化压缩准则**

不微调模型参数，而是优化压缩准则（就是一段自然语言 prompt）。做法是：拿 agent 在有压缩和没压缩两种情况下的轨迹做对比——如果不压缩能做对、压缩了反而做错，说明压缩丢掉了关键信息。这种轨迹级别的对比比简单的奖励信号（比如成功/失败）信息量更大。

具体实现是用一个 LLM 当优化器，用自然语言反馈来迭代改进压缩准则：
1. 在训练集上分别跑不带压缩和带压缩的 agent，挑出那些不带压缩成功、带压缩失败的任务。
2. 对每个这样的任务，把压缩前后的上下文喂给优化器 LLM，让它给出自然语言反馈，相当于一个"文本梯度"。
3. 汇总多条反馈，让 LLM 更新压缩准则。还可以生成多个候选 prompt，挑效果最好的，这叫 utility maximization（UT），主要优化任务奖励。

但光优化奖励可能忽略成本，所以再做一轮：只看带压缩且成功的任务，让 LLM 分析哪些信息是真正用到的，据此把准则改得更精简，这叫 compression maximization（CO），主要优化上下文成本。

**3.4 把上下文压缩蒸馏到小模型**

大 LLM 每次压缩都跑一遍很贵，所以把优化好的压缩器蒸馏到小模型里。用优化了准则的大模型生成压缩后的输出作为监督信号，让小模型用交叉熵损失去学。训练数据只用那些大模型在压缩情况下也能做对的任务。
