## 2026 顶会 Agent 方法

| 工作 | 会议 | 核心贡献 | 开源/复现情况 |
|---|---|---|---|
| **MobileRL** | ICLR 2026 | 用 Difficulty-Adaptive GRPO 做在线多轮 RL；报告 AndroidWorld 80.2%、AndroidLab 53.6% | 框架和模型正在开放，值得优先复现 |
| **K²-Agent** | ICLR 2026 | 将“知道做什么”和“知道怎么操作”拆成高层规划器与低层执行器，并协同演化 | 论文很强，但公开仓库目前不算成熟 |
| **MobileIPL** | ICLR 2026 | 从 action-planning thought tree 构造偏好数据，迭代训练手机 Agent 的推理过程 | 偏模型训练方法 |
| **GUI-Shift** | ICLR 2026 | 从无标注 GUI 转移轨迹做自监督逆动力学与 RL | 适合训练 grounding/execution 模型 |
| **M²-Miner** | ICLR 2026 | 用多 Agent + MCTS 自动挖掘手机 GUI intent–trajectory 训练数据 | 更偏数据生成基础设施 |
| **AutoPlay** | ICLR 2026 | 自动探索 Android App、生成可执行任务与轨迹，再用于 SFT/RL | 同时覆盖 Android 与 Ubuntu |
| **Agent-SAMA** | AAAI 2026 | 用在线构建的 FSM 做规划、状态验证、错误恢复 | 模型无关，比较适合架构研究 |
| **HIM-Agent / PersonalAlign** | ACL 2026 Main | 从长期用户记录中推断隐式意图和个性化偏好 | 手机 Agent 个性化方向 |
| **LearnAct** | ACL 2026 Findings | 从少量操作演示中抽取、检索并复用操作知识 | 代码和 LearnGUI benchmark 已公开 |

最值得优先看的三篇是：

1. **MobileRL：当前训练路线最强**

   ICLR 2026 正式 Poster。核心是 AdaGRPO，包括正样本回放、失败课程过滤、最短路径 reward adjustment。它是真正针对多轮交互式 mobile GUI 环境做在线 RL 的工作。[ICLR 官方页面](https://iclr.cc/virtual/2026/poster/10010890)

2. **K²-Agent：当前 Agent 架构最有启发性**

   将 declarative knowledge——“当前任务接下来应该做什么”，与 procedural knowledge——“具体怎么在 GUI 上完成”，明确解耦。论文报告仅依赖截图和开源 backbone，在 AndroidWorld 达到 76.1%。[ICLR 官方页面](https://iclr.cc/virtual/2026/poster/10011138)

3. **Agent-SAMA：状态建模和错误恢复最直接**

   四个专门 Agent 共同在线构建有限状态机，用它完成规划、执行验证和恢复。报告 AndroidWorld 63.7%，并在跨 App benchmark 上取得更高成功率。[AAAI 官方论文](https://ojs.aaai.org/index.php/AAAI/article/view/40187)

## 2026 顶会 Benchmark 工作

这部分数量甚至比 Agent 方法更多。

| Benchmark | 会议 | 主要解决的问题 |
|---|---|---|
| **FingerTip 20K** | ICLR 2026 | 主动建议、个性化执行、长期用户上下文 |
| **MobileWorld** | ACL 2026 Main | GUI + 用户澄清交互 + MCP 工具调用 |
| **DAC-Bench** | ACL 2026 Main | Android/iOS 条件分支和组合决策任务 |
| **AndroidIntent** | ACL 2026 Main | 隐式意图、用户偏好和主动服务 |
| **A3 / Android Agent Arena** | ACL 2026 Findings | 真实在线 App 与过程状态验证 |
| **LearnGUI** | ACL 2026 Findings | 从少量人类示范中学习 |
| **VenusBench-Mobile** | ICML 2026 Spotlight | 用户中心任务设计和细粒度能力诊断 |
| **ProBench** | AAAI 2026 | 利用准确的过程信息评价 GUI Agent |
| **SMAN-Bench** | ICLR 2026 | 多路径、广告噪声、模糊指令和主动询问 |

其中：

- [FingerTip 20K](https://iclr.cc/virtual/2026/poster/10007512) 收集了约 20K 条真实 Android 长期使用演示，研究“Agent 是否了解这个用户”。
- [MobileWorld](https://aclanthology.org/2026.acl-long.278/) 把 `ask_user` 和 `mcp_call` 纳入手机操作，不再假设用户指令总是完整明确。
- [DAC-Bench](https://aclanthology.org/2026.acl-long.2064/) 覆盖 Android 和 iOS，现有 Agent 在较长的条件分支任务上成功率很低。
- [VenusBench-Mobile](https://openreview.net/forum?id=coHiGZOFtS) 是 ICML 2026 Spotlight，比只看 AndroidWorld 总成功率更强调能力诊断。

## 应当如何理解这批论文

2026 年不是没有顶会，而是研究重心已经发生变化：

- 2023–2024：证明 MLLM 可以操作手机。
- 2024–2025：多 Agent、反思、GUI foundation model。
- 2026：在线 RL、自动数据生成、知识/技能分层、个性化、用户交互和真实动态 App 评测。

如果只筛“既是 2026 顶会、又是真正做手机 GUI Agent 方法、而且值得深入复现”，我建议先读：

**MobileRL → K²-Agent → Agent-SAMA → MobileIPL → LearnAct。**

如果你准备从中找研究切入点，MobileRL 和 K²-Agent 是最直接的两个强 baseline；MobileWorld、DAC-Bench 和 VenusBench-Mobile 则更适合发现现有方法尚未解决的问题。
